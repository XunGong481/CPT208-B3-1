Motivation & Research

The Why

The title "Campus Tour" is derived from a rather typical experience: a new student moves to a campus with a map in hand but still cannot find the dorm area's food locker and has to ask several people for directions. We are convinced that no one feels more lost on campus than in this situation.

Indeed, a short 43-person survey confirms that a new visitor to the campus would find it unfamiliar; new students often lose their way around campus; and some upperclassmen even find some signs or the location of the lockers unclear. The study concluded that campus signage is often inadequate or missing and that mobile maps cannot locate specific entrances or floors. As a result, 72% of respondents rely on asking for directions, while 34% try mobile maps, and if these fail, asking remains the only option.

That is why there is a real need for a system that does not ask for directions: after all, it is much more than just a map—it is a great companion for exploring the campus. With AR, buildings "speak" about themselves; NFC check-ins transform wayfinding into a badge-collecting game; and real-time recommendations bring you closer to points of interest on campus.

Thus, each user—freshman, guest, or returning student—should never lose their way again but instead develop a sense of belonging, pride, and joy in discovering new things on campus.

Academic Papers & Competitor Analysis

📚 Academic Papers (4)

1. Point Cloud Compression for Augmented Reality-Based Campus Navigation Application

Link: https://ieeexplore.ieee.org/document/10213012

✅ 3 Strengths
  • Proposes a point cloud compression method to address wireless channel bandwidth limitations; technical solution is specific
  • Implements a complete technical stack using ARcore + NavMesh + point cloud localisation
  • Published in an IEEE conference, with peer-review endorsement
❌ 3 Gaps
  • Tested only on a single campus (Al-Nahrain University, Baghdad), lacking cross-scenario validation
  • Does not discuss the impact of user age, cultural background, etc. on AR acceptance
  • Does not discuss gamification / entertaining elements; purely functional navigation
2. Development of a Navigation System on the University Territory Using Augmented Reality Technology

Link: https://ieeexplore.ieee.org/document/10238548

✅ 3 Strengths
  • Uses 3DUnity + AR Foundation to achieve cross-platform AR navigation
  • Emphasises "real-time synchronisation" to make users feel the virtual space, enhancing immersion
  • Addresses real pain points of a large campus (106.6 hectares, 26,000 students)
❌ 3 Gaps
  • Covers only a medium-sized campus of 20 buildings; limited scale
  • Does not mention user testing data or sample size
  • Lacks analysis of the specific needs of new students (e.g., registration office, halls of residence, etc.)
3. Smart AR Navigation: Enhancing Campus Wayfinding with Augmented Reality

Link: https://ieeexplore.ieee.org/document/11019523

✅ 3 Strengths
  • Clearly identifies the limitations of traditional maps and signage, i.e., lack of "real-time" and "contextual awareness"
  • Highlights the value of AR on large campuses (e.g., Sri Sairam Engineering College)
  • Positions AR navigation as an extension of "gamification technology"
❌ 3 Gaps
  • Does not provide specific user testing data or evaluation results
  • Does not discuss multi-language support (for international students/visitors)
  • Does not mention accessibility design for people with disabilities (visually impaired/mobility issues)
4. A mobile application for an ecological campus navigation system using augmented reality

Link: https://ieeexplore.ieee.org/document/7297421

✅ 3 Strengths
  • Proposes the concept of an "ecological" navigation system, focusing on sustainability
  • Mobile application lowers the barrier to use
  • Combines environmental concepts with navigation, offering a unique perspective
❌ 3 Gaps
  • Published relatively early (circa 2015); technology may be outdated
  • Does not mention integration with modern AR technologies (e.g., cloud anchors, visual positioning)
  • Lacks quantitative evaluation of user interest/engagement levels

Design Alternatives

Three architectures compared

A Dual-system separated architecture

Build two completely independent backend systems, databases and APIs for new users and visitors. The design is targeted at two types of customers since the functions are completely isolated and do not interfere with each other. However, the cost is too high to maintain.

High Cost Difficult Maintenance
✓ Selected plan
B Single-system multi-module architecture

A unified core system serves both new students and visitors simultaneously through module division. The design has unified data with common core Wanderx Campus Core System. Additionally, the cost is available and is easy to maintain. However, it is necessary isolation to ensure module isolation to avoid interference between the two types of user functions.

Balanced Cost-Effective
C Pure backend-free static architecture

Completely relying on third-party services, without building a self-owned backend. The design is easy to develop and unnecessary to maintain with servers. The front end directly calls the map API with none user database or real-time positioning. Hence, the functions are strictly limited and cannot meet users core needs.

Limited Functions No Real-time Data

Why choose plan B?

🎯
User-Centric

Since there are two personas, the system can provide different modules for different users, such as AR navigation and offline availability.

💰
Cost-Effective

As a student project, the system ensures the complete functionality and controls the costs in the reasonable range.

🏗️
Clear Architecture

The system applies the unified database and the data is synchronized in real time, which ensures data consistency and supports users requirements immediately. Additionally, it provides a unified web entry to reduce usage barriers.

From this perspective, the best tradeoff between functional completeness, maintenance costs, and user experience is found in the single-system, multi-module approach.

While it eliminates the large expense of a two-system approach, it simultaneously mitigates the shortcomings of a backend-less solution. A clear separation between modules, unified by a single database, allows us to continue serving incoming students and visitors with limited resources.

Thus, the single-system multi-module approach best captures the design ethos of the project—"practical, usable, and evolvable."

Final Reflection

Social/Ethical Impact: The WanderX project emphasizes accessibility and inclusivity, ensuring all users can equally access campus navigation services. We considered the needs of different user groups, including new students, visitors, and users with special needs. At the same time, we value user privacy, ensuring the secure use of location data without excessive collection or misuse of user information.

AI Usage: We used various AI tools to assist with project development, including image generation tools for creating concept maps, natural language processing tools for analyzing user feedback, and machine learning models for optimizing AR positioning algorithms. We ensured all AI-generated content was manually verified, and we followed relevant ethical guidelines when using AI tools.
